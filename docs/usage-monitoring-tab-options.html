Usage Monitor TAB / Content Options

用量监测不再像字段清单，改成可决策的能耗驾驶舱。

本页给出 4 种 HTML 方案。共同前提：移除右侧字段面板，把空间还给核心图表；贡献热力图采用横向大画布；内容从“有哪些字段”转向“哪里异常、谁贡献、成本为什么变化、下一步做什么”。

调整原则

  • 01去掉右侧字段面板：字段配置移入筛选器或抽屉，默认不占用监测视野。
  • 02贡献热力图变宽：作为首屏主图，覆盖 52 周或 24 小时 x 7 天。
  • 03加解释层：不仅显示 token、请求、成本，还要显示峰值原因、异常责任归因。
  • 04保留可行动入口：预算预警、限流建议、导出报表、查看明细放在顶部或卡片内。

Finance / Explainability

C. 成本瀑布 + 预算风险

适合管理视角：核心问题不是“用了多少”，而是“为什么多花、还能花多久、该在哪里节省”。瀑布图解释成本增减，风险卡片给出阈值和动作。

适合选择如果目标用户是负责人、财务或平台 owner，需要月度控制和预算预测。

可混搭的其他呈现形式

如果不想一次押注单一大图，可以把这些模块作为 A 的二级切换或点击热力格后的详情层。它们都不需要右侧字段面板，字段筛选统一放到顶部筛选器、保存视图或按需抽屉中。

使用方式首版只选 1-2 个补充模块，避免监测 TAB 重新变成数据堆叠页。

小时 x 星期热力图

展示 7 天 24 小时峰值，适合发现夜间批处理和工作时段拥堵。

模型占比环带

用同心环显示输入、输出、缓存、重试占比，快速看成本结构。

预算燃尽 + 预测

显示实际消耗、理想消耗和月底预测线，给出撞线日期。

异常任务排行榜

按成本突增、失败重试、长输出、低缓存命中排序，直接进入明细。

贡献者小倍图

每个用户或项目一张小趋势图，便于比较“谁在持续增长”。

请求漏斗

从请求、成功、重试、失败到人工介入，适合观察浪费发生在哪一层。

模型切换模拟器

拖动场景占比，预估从高阶模型切到 mini 后的成本和质量风险。

日历注释层

在宽热力图上叠加发布、配置变更、批处理事件，解释峰值原因。

选择建议

如果只能落一个版本，建议先选 A：它符合“移除右侧字段面板、贡献热力图更宽”的明确诉求，同时不会改变用户对监控 TAB 的认知。B/C/D 可以作为二级视图或后续迭代。

首版主线A 的大热力图 + KPI + 异常摘要，最稳妥。
增强分析把 B 的流向图放进“查看归因”抽屉。
管理视角将 C 的预算风险作为顶部告警卡常驻。
团队治理D 适合作为“贡献者”维度的切换模式。
字段面板替代字段选择改为顶部筛选器 + 保存视图。
可行动指标每个异常都给出责任归因、影响范围、建议动作。
