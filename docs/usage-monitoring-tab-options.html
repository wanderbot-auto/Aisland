Aisland Cyber-Minimalist / Usage Monitoring

把用量监测收成一块安静、可信、能行动的仪表板。

这版不再做泛化 BI 页面，而是贴合 Aisland 的黑色玻璃、细描边、青色状态光和紧凑 macOS 工具感。核心判断：用量页不是“字段浏览器”，而是帮用户在几秒内回答三件事——今天有没有异常、异常从哪里来、下一步怎么处理。

这版的取舍

  • 01移除右侧字段面板默认视野只保留监测、解释、行动。字段选择放到顶部筛选和“保存视图”。
  • 02贡献热力图变成主画布横向 52 周宽图占据首屏，保持 Aisland 的轻密度信息风格。
  • 03解释少而准用 3 个 insight 卡回答峰值、来源、建议，而不是堆更多图。
  • 04延续主题语言黑底、玻璃卡、单一青色主强调，异常只用小面积琥珀/红色。

C / 预算与管理

Budget Burn：少数 owner 需要的成本解释

把“用了多少”翻译成“还能花多久、为什么变贵、哪里能省”。它适合二级视图或风险告警卡，不适合占据默认监控首屏。

实用点给预算 owner 直接动作：限制批处理、切模型、开缓存，而不是只看账单。

D / 团队治理

Team Matrix：贡献者维度，不做默认入口

当用户需要分摊成本或培训团队时，把宽热力图切成贡献者矩阵。它信息密度高，但默认展示会让页面变重，所以建议作为维度切换。

实用点发现“高成本低成功率”的自动化任务，以及哪个团队需要优化 prompt / 缓存 / 模型选择。

建议落地顺序

选择 A 作为实现目标：它贴合 Aisland 的主题和当前代码结构，也最符合“美观、实用、简洁”。B/C/D 不丢弃，但降级为从 A 进入的二级解释视图。

第一步SwiftUI 中移除字段侧栏心智：用顶部筛选、保存视图、详情抽屉替代。
第二步把贡献热力图放在首屏最宽区域，保留现有日趋势作为下方辅助。
第三步新增三条解释卡：峰值原因、贡献来源、建议动作，避免用户只看到图。
第四步异常格点击后再进入 B 的归因抽屉；预算风险只在接近阈值时显示。
第五步颜色严格收敛：青色为主，绿色稳定，琥珀预警，红色只用于异常峰值。
结论先做 A，不做大而全；让用量页像 Aisland 的监控面板，而不是后台报表。
