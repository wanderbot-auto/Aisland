不用热力图也能表达 Token 消耗

6 种更适合统计页的用量图形方案

热力图擅长看“哪里热”，但不擅长表达总量走势、模型结构、成本归因和会话爆发。下面每个方案都用同一套 Aisland 深色玻璃风格模拟，方便直接挑选后落到 SwiftUI。

当前偏好：方案 B + C 主视图用日期卡片柱状日历；右侧保留模型贡献 Pareto。悬浮某一天时，浮层显示当天模型构成和 24 小时时间热力图。
精选组合 A 趋势面积 B 日期卡片 C 模型排行 D 环形配额 E 会话时间线 F 来源流向

选择建议

如果统计页只放一个主图：选 A。若想看“谁花得最多”：选 C。若重点是预算/配额压力：选 D。若要解释某天为什么暴涨：选 E 或 F。

不建议继续强化热力图的场景

当用户主要关心 token 总量、成本、模型占比、趋势变化时，热力图的信息密度高但判断成本也高，容易看起来像噪点矩阵。
